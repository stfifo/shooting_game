## 간단 설명과 사용법

- HTML/CSS: 화면 레이아웃과 스타일
- JavaScript: 게임 로직, 렌더링, 입력 처리
- 사용법
  - 화살표로 로켓 이동 가능
  - z: bomb 터트리기 가능 (화면 하단에 bomb이 있어야 가능함)
  - x: 스킬 실행 가능. 화면 하단의 스킬 bar가 다차면 스킬 실행 가능함
  - p: 게임 나가기
  - enter: 게임 실행

## 클로드 기법
hook와 custom slash commands를 사용했다.

### hook
hooks를 사용하여 워크플로우 자동화하는 방법은 [클로드 공식 문서](https://code.claude.com/docs/ko/hooks-guide)를 참고했다.  
hook를 만들기 위해 `settings.json`에 `hooks` 블록을 추가하였다.
- `Notification`
    알림 hook를 만들어 클로드가 입력이 필요할 때 알림을 띄우도록 자동화하였다.
    - `"matcher": ""` 와 같이 matcher가 비어있으면 모든 유형에서 알림이 띄운다. 특정 이벤트에서만 발생하도록 하려면 아래 값 중에서 matcher를 지정하면 된다
  
    | Matcher | 발생 시점 |
    |---|---|
    | `permission_prompt` | Claude가 도구 사용을 승인하도록 요청할 때 |
    | `idle_prompt` | Claude가 완료되고 다음 프롬프트를 기다릴 때 |
    | `auth_success` | 인증이 완료될 때 |
    | `elicitation_dialog` | MCP 서버가 유도 양식을 열 때 |
    | `elicitation_complete` | MCP 유도 양식이 제출되거나 닫힐 때 |
    | `elicitation_response` | MCP 유도 응답이 서버로 다시 전송될 때 |

- `PostToolUse`
    파일 수정 hook를 만들어 클로드가 파일을 수정할 때마다 자동으로 백업하도록 자동화하였다.
    - `"matcher": "Edit|Write|MultiEdit"` 으로 지정하여 파일 편집 및 생성 도구가 실행된 직후에만 동작하도록 하였다.
    - PowerShell 명령어로 `backups/` 폴더를 생성하고, 백업 파일명 앞에 순서 번호(001, 002, ...)와 타임스탬프를 붙여 저장하도록 하였다.
    - 백업 파일명 예시: `001_game_20250520_143022.html`
- `Stop`
    작업 완료 hook를 만들어 클로드가 작업을 완료했을 때 안내 메시지를 출력하도록 자동화하였다.
    - `Stop` hook는 `matcher` 없이 사용하며, 클로드가 응답을 마칠 때마다 자동으로 발동된다.
    - PowerShell 명령어로 터미널에 `작업 완료! html을 열어서 확인하세요.` 메시지를 출력하도록 하였다.

### custom slash commands
custom slash commands를 사용하여 자주 쓰는 작업을 단축 명령어로 등록하였다.  
`.claude/commands/` 폴더에 마크다운 파일을 추가하면 클로드 코드에서 `/명령어`로 호출할 수 있다.

- `/design`
    게임의 시각적 디자인 완성도를 높이는 명령어이다.
    - 배경은 세로 스크롤되는 지구/우주 맵으로 구현하며, 시간이 지날수록 지구에서 화성, 목성 등 우주의 범위가 확장되도록 하였다.
    - 플레이어 전투기와 적 전투기/폭격기는 1990년대 느낌의 픽셀 아트로 표현하도록 하였다.
    - 총알, 폭발, 피격 이펙트는 픽셀 느낌의 파티클과 함께 구현하도록 하였다.
    - 보스 등장 시 별도의 경고 연출을 구현하도록 하였다.
    - 임의로 판단하기 어려운 부분이 있으면 즉시 멈추고 질문하도록 하였다.

- `/logic`
    게임 로직과 밸런스를 개선하는 명령어이다.
    - 적은 곡선/편대 패턴으로 등장하도록 하였다.
    - 특수 공격은 획득 점수가 특정 점수 이상일 때 발동 가능하며, 파이어볼처럼 전투기 화력이 커지는 방식으로 구현하도록 하였다.
    - 스테이지 마지막에 보스가 등장하고, 체력 구간별로 패턴이 변화하는 다중 페이즈를 가지도록 하였다.
    - 적 격추 시 파워업, 폭탄, 목숨 아이템이 일정 확률로 드롭되도록 하였다.
    - 아이템 획득 시 7초 동안 플레이어 전투기 수가 증가하는 편대 아이템을 추가하도록 하였다.
    - 임의로 판단하기 어려운 부분이 있으면 즉시 멈추고 질문하도록 하였다.

- `/uiux`
    게임의 UI/UX 완성도를 높이는 명령어이다.
    - HUD(점수, 목숨, 레벨)는 게임 플레이를 방해하지 않는 위치에 표시하도록 하였다.
    - 시작 → 플레이 → 일시정지 → 게임오버/클리어 화면 흐름이 자연스럽게 연결되도록 하였다.
    - 키보드 입력에 대한 시각적 피드백을 추가하도록 하였다.
    - 조작법 안내는 시작 화면에서 표시하도록 하였다.
    - 임의로 판단하기 어려운 부분이 있으면 즉시 멈추고 질문하도록 하였다.

- `/refactor`
    게임 코드 구조와 가독성을 개선하는 명령어이다.
    - 기능별로 함수를 분리하고, 하나의 함수가 하나의 역할만 하도록 정리하였다.
    - 매직 넘버는 상수로 추출하고 변수명은 역할이 명확하게 드러나도록 작성하도록 하였다.
    - 중복 코드는 제거하고 공통 로직은 함수로 묶도록 하였다.
    - 주요 함수와 복잡한 로직에는 한국어 주석을 추가하도록 하였다.
    - 동작 변경 없이 구조 개선만 하도록 제한하였다.
    - 임의로 판단하기 어려운 부분이 있으면 즉시 멈추고 질문하도록 하였다.
