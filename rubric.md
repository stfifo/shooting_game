# 평가 루브릭 (Strikers 1945+ Claude Code 프로젝트)

## 평가 방식

### 방식 1 — 다차원 기준별 점수 (3개 차원, 각 1~5점)

각 케이스를 아래 세 차원으로 독립 평가한다. 최종 방식 B 점수는 세 차원의 **평균**을 반올림한 값(1~5 정수)으로 산출한다.

| 차원 | 의미 |
|---|---|
| **Correctness** | 구현이 버그 없이 의도대로 동작하는가 |
| **Completeness** | 요구사항 전체를 빠짐없이 구현했는가 |
| **Code Quality** | 기존 코드베이스와 일관성 있게 작성되었는가 |

#### Correctness 기준

| 점수 | 기준 |
|---|---|
| 5 | 버그 없음. 엣지 케이스(경계값, 동시 발생 등)까지 모두 올바르게 처리됨 |
| 4 | 핵심 동작은 정확하나 사소한 엣지 케이스 하나가 누락됨 |
| 3 | 핵심 동작은 하나, 일부 시나리오에서 잘못 동작하거나 미완성 조건 분기가 있음 |
| 2 | 동작하지만 중요한 조건에서 버그 발생, 수정 없이는 실사용 어려움 |
| 1 | 잘못된 로직 또는 구현 오류로 게임이 크래시되거나 아예 동작하지 않음 |

#### Completeness 기준

| 점수 | 기준 |
|---|---|
| 5 | 요청한 기능 전체 구현. 관련 파일(상태변수, HUD, 초기화 등) 모두 일관되게 갱신됨 |
| 4 | 요청 기능의 90% 이상 구현. 시각/UI 표시 등 부가 사항 1~2개 미반영 |
| 3 | 핵심 기능은 구현됐으나 연관 컴포넌트(HUD 반영, 상태 초기화 등) 일부 누락 |
| 2 | 요청의 50% 이하 구현. 핵심 동작 자체가 빠져 있음 |
| 1 | 요청을 잘못 이해하거나 무관한 코드를 변경함 |

#### Code Quality 기준

| 점수 | 기준 |
|---|---|
| 5 | 기존 명명 규칙·스타일 완벽히 준수. 불필요한 코드 없음. 전역 스코프 오염 없음 |
| 4 | 스타일 대체로 일관. 사소한 중복 또는 미세한 네이밍 불일치 1~2건 |
| 3 | 동작하나 리팩터링이 필요한 중복·하드코딩·불명확한 변수명 다수 |
| 2 | 기존 코드 패턴과 크게 불일치하거나, 과도한 전역 변수 추가 |
| 1 | 코드 구조 훼손, 기존 함수 충돌, 또는 파일 의존 순서 위반 |

### 방식 2 — 홀리스틱 루브릭 (1~5점, 레벨별 기준 명시)

케이스 전체를 하나의 점수로 종합 평가한다.

| 점수 | 기준 |
|---|---|
| **5** | 요청 기능을 완벽하게 구현. 엣지 케이스 처리, 관련 컴포넌트 갱신(HUD·상태·초기화), 코드 일관성 모두 충족. 추가 수정 없이 게임에 즉시 통합 가능한 수준 |
| **4** | 핵심 기능은 올바르게 동작하며 실용적으로 완성도가 높음. 시각 표시 미비, 엣지 케이스 1개 누락, 사소한 코드 스타일 불일치 등 경미한 결함이 1~2건 존재 |
| **3** | 핵심 기능은 구현됐으나 요구사항의 일부(연관 UI 반영, 상태 변수 추가 등)가 빠져 있음. 게임에서 동작하지만 완전하지 않아 추가 작업이 필요 |
| **2** | 기능 의도는 이해했으나 구현에 버그가 있거나 요구사항의 절반 이하만 충족. 수정 없이는 정상 동작하지 않을 가능성이 높음 |
| **1** | 요청을 잘못 이해하거나 구현이 완전히 실패. 버그로 게임이 정상 작동하지 않거나 무관한 코드를 변경함 |
