## 빠른 참고 — 평가 기준

### 방식 1 · Correctness (정확성)
| 점수 | 기준 |
|---|---|
| 5 | 버그 없음. 엣지 케이스(경계값, 동시 발생 등)까지 모두 올바르게 처리됨 |
| 4 | 핵심 동작은 정확하나 사소한 엣지 케이스 하나가 누락됨 |
| 3 | 핵심 동작은 하나, 일부 시나리오에서 잘못 동작하거나 미완성 조건 분기가 있음 |
| 2 | 동작하지만 중요한 조건에서 버그 발생, 수정 없이는 실사용 어려움 |
| 1 | 잘못된 로직 또는 구현 오류로 게임이 크래시되거나 아예 동작하지 않음 |

### 방식 1 · Completeness (완성도)
| 점수 | 기준 |
|---|---|
| 5 | 요청한 기능 전체 구현. 관련 파일(상태변수, HUD, 초기화 등) 모두 일관되게 갱신됨 |
| 4 | 요청 기능의 90% 이상 구현. 시각/UI 표시 등 부가 사항 1~2개 미반영 |
| 3 | 핵심 기능은 구현됐으나 연관 컴포넌트(HUD 반영, 상태 초기화 등) 일부 누락 |
| 2 | 요청의 50% 이하 구현. 핵심 동작 자체가 빠져 있음 |
| 1 | 요청을 잘못 이해하거나 무관한 코드를 변경함 |

### 방식 1 · Visual Design (시각 디자인)
| 점수 | 기준 |
|---|---|
| 5 | 시각 효과(FX 텍스트·색상·애니메이션)가 기존 픽셀 아트 스타일과 완벽히 일치. 플레이어에게 명확한 시각적 피드백 제공. HUD 정보가 가독성 높게 배치됨 |
| 4 | 핵심 시각 요소는 구현됐으나 색상·크기·위치 등 세부 표현에 사소한 불일치 1~2건 존재 |
| 3 | 기능은 동작하나 시각 피드백이 불명확하거나 HUD 표시가 부자연스러움. 추가 디자인 작업이 필요한 수준 |
| 2 | 시각 요소가 누락됐거나 기존 픽셀 아트 스타일과 크게 불일치하여 플레이어가 상태를 인지하기 어려움 |
| 1 | 시각 구현 없음 또는 기존 렌더링 코드와 충돌하여 화면이 깨지거나 아무것도 표시되지 않음 |

### 방식 2 · 종합 Rubric
| 점수 | 기준 |
|---|---|
| 5 | 요청 기능을 완벽하게 구현. 엣지 케이스 처리, 관련 컴포넌트 갱신(HUD·상태·초기화), 코드 일관성 모두 충족. 추가 수정 없이 게임에 즉시 통합 가능한 수준. 게임이 정상적으로 클리어됨. |
| 4 | 핵심 기능은 올바르게 동작하며 실용적으로 완성도가 높음. 시각 표시 미비, 엣지 케이스 1개 누락, 사소한 코드 스타일 불일치 등 경미한 결함이 1~2건 존재. 게임 난이도가 레벨을 반영하지 않음. |
| 3 | 핵심 기능은 구현됐으나 요구사항의 일부(연관 UI 반영, 상태 변수 추가 등)가 빠져 있음. 게임에서 동작하지만 완전하지 않아 추가 작업이 필요. UI가 사용자에게 명확하게 보이지 않음. |
| 2 | 기능 의도는 이해했으나 구현에 버그가 있거나 요구사항의 절반 이하만 충족. 수정 없이는 정상 동작하지 않을 가능성이 높음. 게임 클리어해도 정상적으로 종료 안 되거나 게임이 영원히 실행됨. |
| 1 | 요청을 잘못 이해하거나 구현이 완전히 실패. 버그로 게임이 정상 작동하지 않거나 무관한 코드를 변경함. |
