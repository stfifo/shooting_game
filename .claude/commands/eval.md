현재 프로젝트의 Claude Code 출력 결과물을 평가하고 보고서를 작성해줘.

## 평가 목적
Claude Code로 만든 프로젝트를 대상으로 출력 품질을 1) 기계(LLM-as-Judge) 2) 사람 두 번 평가하고, 둘의 일치도를 Cohen's Kappa로 측정한다.

## 평가 방식 추천 및 선택
아래 3가지 방식 중 적합한 2가지를 추천하고, 이유를 설명해라. 추천 후 사용자에게 어떤 방식으로 진행할지 확인해라.

- **방식 A**: 기준 없이 직관적으로 1~5점
- **방식 B**: Helpfulness, Correctness, Completeness 등 몇 가지 기준을 만들어 1~5점
- **방식 C**: Rubric을 만들어 1점~5점 각각의 기준을 명시하고 평가

## 평가 진행 순서
1. 평가 방식 2가지를 확정해라 (사용자 확인 후 진행)
2. 평가 대상 케이스 목록을 정리해라 (10개 권장)
3. 각 케이스에 대해 사람이 먼저 점수를 매길 수 있도록 입력 인터페이스를 제공해라
   - 반드시 사람 점수를 먼저 입력받고, 기계 평가는 그 이후에 실행해라
4. 사람 점수 입력 완료 후 Claude API로 동일 케이스를 자동 평가해라
5. `sklearn`의 `cohen_kappa_score`로 일치도를 측정해라
   - `weights=None`과 `weights='quadratic'` 두 가지 모두 계산해라
   - 두 결과의 차이와 의미를 설명해라
6. 평가 결과를 점수 비교표로 출력해라
7. 아래 보고서 작성 지침에 따라 보고서를 작성해라
